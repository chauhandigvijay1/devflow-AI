## Account & Authentication

### How do I create an account?

Navigate to the [signup page](https://devflow-ai-client.netlify.app/signup) and fill in your name, username, email, and password. 

> [!IMPORTANT]
> Your password must be at least 8 characters long and include at least one uppercase letter, one lowercase letter, and one digit.

Alternatively, use the quick register option which only requires your name, email, and password.

### Can I use my email or username to log in?

Yes, both are supported. The login endpoint accepts either an `email` or `identifier` field. The identifier can be either your email address or your username.

### I forgot my password. What should I do?

Visit the [forgot password page](https://devflow-ai-client.netlify.app/forgot-password) and enter your email address. If an account exists with that email, you will receive a password reset link.

> [!WARNING]
> The password reset link expires in 15 minutes. If no email is received, check your spam folder or verify that the email address is correct.

### How do I delete my account?

Go to **Settings → Danger Zone → Delete Account**. Type "DELETE" in the confirmation field and click confirm. Your account will be soft-deleted immediately. 

> [!NOTE]
> Chat histories are preserved securely, but your email and username are freed for reuse.

### Can I recover a deleted account?

Not through the user interface. Accounts are soft-deleted (`isDeleted: true`) and can be restored by toggling the flag in MongoDB.

> [!TIP]
> Contact the project maintainer at [chauhandigvijay669@gmail.com](mailto:chauhandigvijay669@gmail.com) for admin-level account recovery.

## AI Chat

### What AI model does DevFlow AI use?

DevFlow AI uses **Llama 3.1 8B** served through **Groq Cloud's LPU hardware** for ultra-low-latency inference. The specific model identifier is `llama3-8b-8192`. You can configure a different model via the `AI_MODEL` environment variable.

### How does the streaming work?

The AI chat endpoint (`POST /api/ai/prompt`) uses **Server-Sent Events (SSE)** to stream tokens one at a time as they are generated by Groq. The client reads the stream using the Fetch API's `ReadableStream` reader and renders tokens incrementally. This provides a fast, real-time typing effect.

```javascript
// Example: Consuming the SSE stream on the client
const response = await fetch('/api/ai/prompt', { method: 'POST', body });
const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  console.log(decoder.decode(value));
}
```

### What is the code explanation endpoint?

The code explanation endpoint (`POST /api/ai/explain`) is a non-streaming, single-turn endpoint that takes a code snippet and an optional language identifier, returning a plain-text explanation. It operates using a specific system prompt: *"You are a senior developer who explains code simply."*

### What are the usage limits?

| Plan | Daily Limit | Payload Limit |
|---|---|---|
| **Free** | 20 prompts per day | 8,000 characters per prompt |
| **Pro** | 999 prompts per day (effectively unlimited) | 8,000 characters per prompt |
| **Both** | Code explanation: 50,000 characters | — |

### Why did I get "No response from AI"?

This issue can occur under the following circumstances:
- The Groq API returned an empty response.
- The stream timed out (there is a 60-second limit).
- The prompt was too long (exceeding the 8,000 character limit).
- The `GROQ_API_KEY` is invalid or has exhausted its quota.

> [!TIP]
> Check the server logs for detailed Groq API error messages.

## Pricing & Billing

### How much does Pro cost?

Pro is **₹299/month** (29,900 paise). Payment is handled securely via Razorpay and supports UPI, credit/debit cards, net banking, and wallets.

### Are there any discounts or coupons?

Yes. Two public coupons are available:
- `FREETRIAL` — 100% discount, 7-day Pro access (one-time use per account)
- `OFF50` — 50% discount, 30-day Pro access (one-time use per account)

> [!NOTE]
> The project maintainer may also distribute a secret owner coupon for 100% lifetime discounts.

### How do I apply a coupon?

Enter the coupon code on the billing page before clicking "Upgrade to Pro." The coupon will be validated and the price adjusted accordingly before the Razorpay checkout modal opens.

### Can I cancel my subscription?

Yes. Navigate to **Settings → Billing** and click "Cancel Subscription." Cancellation is immediate, meaning you lose Pro access right away. 

> [!WARNING]
> There are no prorated refunds available upon cancellation.

### Do you offer recurring billing?

Not yet. Pro subscriptions are currently one-time payments with a fixed duration (30 days by default, or the coupon-specific duration). When the duration expires, you will need to manually re-upgrade. Recurring billing is on our roadmap.

### What payment methods are accepted?

Razorpay supports:
- **UPI**: Google Pay, PhonePe, Paytm
- **Cards**: Credit/debit cards (Visa, Mastercard, RuPay)
- **Other**: Net banking, EMI, and wallets

All transactions are processed in INR.

## Security & Privacy

### How are passwords stored?

Passwords are hashed using **bcrypt with 12 salt rounds**. The `password` field has `select: false` in the Mongoose schema, meaning it is never included in query results by default. The pre-save hook only hashes the password when it has been modified.

### How are payment details handled?

Payment details are handled entirely by **Razorpay's secure checkout**. The DevFlow AI server never receives or stores credit card numbers, UPI IDs, or bank account details. The server only receives payment verification data (order ID, payment ID, signature) which is securely validated using HMAC-SHA256.

### Is my chat data private?

Chat data is stored securely in MongoDB Atlas and is associated with your user account. Only you can access your chats (via JWT-authenticated API calls). 

> [!NOTE]
> The AI provider (Groq Cloud) receives prompt text for inference but does not store it beyond the request lifecycle, adhering strictly to their data policy.

### What data is collected?

The application collects minimal essential data:
- Account information (name, email, username)
- Chat messages and AI interactions
- Usage statistics (prompt count, plan status)
- User preferences (settings)

> [!IMPORTANT]
> No analytics or tracking cookies are used. The application does not use Google Analytics, Facebook Pixel, or similar third-party trackers.

### How do you protect against spam accounts?

- **Disposable email blocklist**: Temporary email domains are blocked at registration.
- **Rate limiting**: Login and forgot-password endpoints are throttled to 20 requests per 15 minutes.
- **Duplicate detection**: Email and username uniqueness is strictly enforced at the database level.

## Best Practices

To get the most out of your DevFlow AI experience:
- **Keep Prompts Concise**: While you have an 8,000-character limit, clearer and shorter prompts yield faster and more accurate AI responses.
- **Secure Your Account**: Always use a unique password and avoid sharing your account credentials.
- **Monitor Your Usage**: If you are on the Free plan, keep an eye on your 20-prompt daily limit to ensure you have capacity when you need it most.
